torchkit.logger¶

The Logger class is a simple wrapper over Pytorch’s SummaryWriter. You can use it to log losses, learning rates and metrics. For more fine-grained control, you should use the log_scalar, log_dict_scalars and log_image methods.

class torchkit.logger.Logger(log_dir: str, force_write: bool = False)¶

A Tensorboard-based logger.

__init__(log_dir: str, force_write: bool = False)¶

Constructor.

Parameters

  • log_dir – The directory in which to store Tensorboard logs.

  • force_write – Whether to force write to an already existing log dir. Set to True if resuming training.

log_dict_scalars(dict_scalars: Mapping[str, float], global_step: int, prefix: str) → None¶

Log a dictionary of scalars.

Parameters

  • scalar – A scalar torch.Tensor or float.

  • global_step – The training iteration step.

  • prefix – A prefix to prepend to each logged scalar.

log_image(image: Union[torch.Tensor, numpy.ndarray], global_step: int, prefix: str, name: str = '', dataformat: str = 'CHW') → None¶

Log an image or batch of images.

Parameters

  • image – A torch.Tensor or numpy array. Can be a single image or a batch of images. If a torch.Tensor is provided, the format should be CHW. If a numpy array is provided, the format should be HWC.

  • global_step – The training iteration step.

  • prefix – A prefix to prepend to the logged image(s).

  • name – The name of the logged image(s).

  • dataformat – How the image data is stored. The default value is CHW assuming the input is a torch.Tensor. If feeding numpy arrays, set it to HWC.

log_loss(losses: Union[Mapping[str, float], float], global_step: int) → None¶

log_metric(metrics: Mapping[str, Any], global_step: int, metric_name: str) → None¶
