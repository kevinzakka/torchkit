torchkit.losses¶

torchkit.losses.one_hot(y: torch.Tensor, K: int, smooth_eps: float = 0) → torch.Tensor¶

One-hot encodes a tensor with optional label smoothing.

Parameters

  • y – A tensor containing the ground-truth labels of shape (N,), i.e. one label for each element in the batch.

  • K – The number of classes.

  • smooth_eps – Label smoothing factor in [0, 1] range.

torchkit.losses.cross_entropy(logits: torch.Tensor, labels: torch.Tensor, smooth_eps: float = 0, reduction: str = 'mean') → torch.Tensor¶

Cross-entropy loss with support for label smoothing.

Parameters

  • logits – A FloatTensor containing the raw logits, i.e. no softmax has been applied to the model output. The tensor should be of shape (N, K) where K is the number of classes.

  • labels – A rank-1 LongTensor containing the ground truth labels.

  • smooth_eps – The label smoothing factor in [0, 1] range.

  • reduction – The reduction strategy on the final loss tensor.

Returns

If reduction is none, a 2D tensor. If reduction is sum, a 1D tensor. If reduction is mean, a scalar 1D tensor.

torchkit.losses.huber_loss(input: torch.Tensor, target: torch.Tensor, delta: float, reduction: str = 'mean') → torch.Tensor¶

Huber loss with tunable margin [1].

This is a more general version of PyTorch’s torch.nn.functional.smooth_l1_loss that allows the user to change the margin parameter.

Parameters

  • input – A FloatTensor representing the model output.

  • target – A FloatTensor representing the target values.

  • delta – Given the tensor difference diff, delta is the value at which we incur a quadratic penalty if diff is at least delta and a linear penalty otherwise.

  • reduction – The reduction strategy on the final loss tensor.

Returns

If reduction is none, a 2D tensor. If reduction is sum, a 1D tensor. If reduction is mean, a scalar 1D tensor.

References

[1]: Wikipedia Huber Loss, https://en.wikipedia.org/wiki/Huber_loss
